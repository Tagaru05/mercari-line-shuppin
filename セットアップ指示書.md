# セットアップ指示書（実行者＝このPCのClaude Code）

あなた（Claude Code）はこれから、このPCに「LINE写真→メルカリ下書き全自動ライン」を構築する。
ユーザーはAI初心者。**専門用語を使わず、ユーザー操作は1つずつ・スクショ例えつきで案内すること。**
この指示書は、先行環境（タガル環境・2026-07-17）で実証済みの手順の完全版である。

## 原則（最初に読む）

- **原則、画面操作・CLI・ファイル作成・設定はすべてあなた（Claude Code / Claude in Chrome）がやる。** ユーザーに画面を見て判断させない
- ユーザーに頼むのは、あなたに物理的にできない4つだけ：**各サービスのログイン／スマホに届く認証コードの入力／スマホでのQR友だち追加／許可ボタン**
- **秘密の文字列（トークン等）をチャットに出させない・出さない**。画面から読み取って専用の.env.tmpに書き込むのはあなたの作業
- 各ステップ完了ごとに実物確認（curl・ファイル存在・応答テスト）をしてから次へ進む
- 途中で失敗しても壊れない。同じステップをやり直せばよい
- ログイン画面・認証コード入力欄だけはブラウザ拡張が触れない仕様（正常）。そこだけユーザーに渡す

## Step 0：前提確認（5分・**ここで全部そろうまで先に進まない**）

4. **常駐監視（Monitor）ツールが使えるか確認**。使えない環境なら、全自動は諦めて**手動合図運用**（ユーザーが「仕込みして」と言ったら処理）に切り替える旨をユーザーに説明して合意を取る
5. Chromeで jp.mercari.com にログイン済みか確認（未ログインならユーザーに）
6. ChatGPTログイン確認
7. 作業フォルダを作る：`~/Desktop/メルカリ出品ライン/`（このキットの中身をそこへコピー）
8. **設定メモを作る**：`~/Desktop/メルカリ出品ライン/config.md` を作成し、以降のステップで決まる値（Worker URL・KV id・GPTsリンク・都道府県）を**必ずここに記録していく**（翌日以降のセッションはこのファイルを読んで動く）

## Step 1：切り抜き係（rembg）の導入（5分・全部あなた）

**必ずvenv（仮想環境）方式で入れる**。最近のMacは `pip3 install` を直接叩くと externally-managed-environment エラーで失敗する：

```bash
python3 -m venv ~/Desktop/メルカリ出品ライン/venv
~/Desktop/メルカリ出品ライン/venv/bin/pip install rembg onnxruntime scipy pillow numpy
~/Desktop/メルカリ出品ライン/venv/bin/python -c "import rembg; print('OK')"
```
以降、白抜き実行は必ず `venv/bin/python` を使う（config.mdにもその旨を記録）。
初回実行時にモデル（約180MB×2）が自動DLされる。isnet-general-use を使う。

## Step 2：無料の受付サービス（Cloudflare）を作る（15分）

1. ユーザーに案内：https://dash.cloudflare.com/sign-up で無料アカウント作成（メール認証まで）
2. あなたが実行：`npm install -g wrangler`（権限エラー時は以降すべて `npx wrangler` で読み替え）→ `wrangler login`（ブラウザが開くので許可ボタンはユーザー）
3. あなたが実行（コマンドが失敗したらバージョン差なので `wrangler --help` で現構文を確認して読み替える）：
```bash
cd ~/Desktop/メルカリ出品ライン/worker
wrangler kv namespace create UKETSUKE   # 出力されたidを控え、config.mdに記録
```
4. `wrangler.toml` を作成（nameはユーザー名入りでユニークに。例 line-mercari-hanako）：
```toml
name = "line-mercari-（ユーザー名ローマ字）"
main = "worker.js"
compatibility_date = "2026-07-01"

[[kv_namespaces]]
binding = "UKETSUKE"
id = "（3で出たid）"
```
5. `wrangler deploy` → 出たURL（https://～.workers.dev）を**config.mdに記録**
6. 閲覧キーを作る：
```bash
openssl rand -hex 16 > .list_key
wrangler secret put LIST_KEY < .list_key
```
7. 実物確認：`curl https://（URL）/` が「稼働中（持ち主登録: 未…）」を返すこと
8. 依存のバージョンを `config.md` に記録（wrangler/rembgの実測バージョン。将来の再現用）。npmのグローバル導入で権限エラーが出たら `npx wrangler` 方式に切り替える

## Step 3：LINE公式アカウント（受付窓口）を作る（★ここは原則あなた=Claude Codeが画面代行する）

**重要な方針（2026-07-18確立）**：ここが一番むずかしい所だが、**画面操作はあなた（Claude in Chrome）が代行する**。
ユーザーにやってもらうのは、**あなたには物理的にできない2つだけ**：
- **本人のスマホに届く認証コードの入力**（アカウント作成時のSMS/メール認証）
- **スマホでQRを読んで友だち追加**（Step 3-7）

それ以外（画面遷移・ボタン・トークン発行のコピー・貼り付け・設定）は全部あなたがChrome操作でやる。
「ユーザーに画面を見て判断させる」進め方は絶対にしない。まず tabs_context / navigate で操作を始め、詰まったポイントだけユーザーに頼む。

> 画面の文言・ボタン位置は変わることがある。目的（Messaging API有効化／トークン発行／Webhook URL登録／応答メッセージOFF）を頼りに探し、迷ったらWeb検索で現行UIを確認。断定手順に固執しない。
> ログイン画面と認証コード入力欄だけはブラウザ拡張が触れない仕様（正常）。そこはユーザーに渡す。

1. **あなたがChromeで** https://entry.line.biz/ を開き、公式アカウント作成を進める（名前は「出品おてつだい」等を提案）。
   - **メール/電話の認証コード入力が出たら**：ユーザーに「スマホ（またはメール）に届いたコードを教えてください」と頼み、あなたが入力欄に打ち込む。ログイン自体が拡張で無理な画面なら、その画面だけユーザーにログインを頼み「入れたら教えて」で受け取る
2. **あなたがChromeで** https://developers.line.biz/console/ を開き、そのアカウントのチャネルで **Messaging API を有効化**（画面操作はあなた）
3. **チャネルアクセストークン（長期）を発行**（発行ボタンはあなたが押す）→表示された値を、あなたが `worker/.env.tmp`（作成時 `chmod 600`）の `LINE_CHANNEL_ACCESS_TOKEN=` 行に書き込む。※画面から値を読み取ってファイルに書くのはあなたの作業。ユーザーにコピペさせる必要はない（拡張が値を読めない場合のみ、コピーを押して.env.tmpに貼るところだけ頼む）。**チャットには絶対に出さない**
4. あなたが実行：`grep '^LINE_CHANNEL_ACCESS_TOKEN=' worker/.env.tmp | cut -d= -f2- | wrangler secret put LINE_CHANNEL_ACCESS_TOKEN`
5. 同様にチャネルシークレット（チャネル基本設定タブ）も、あなたが `worker/.env.tmp` 経由で `LINE_CHANNEL_SECRET` に設定
6. **あなたがChrome代行で**Webhook設定（実証済み）：
   - Messaging API設定タブ → Webhook URL に `https://（WorkerURL）/webhook` を入力 → Update → Verify（Success確認）→ Use webhook トグルON
   - manager.line.biz → 応答設定 → **応答メッセージOFF**（あいさつはONのまま）
7. **持ち主登録（v0.3内蔵フロー）**：登録が済むまで写真は一切保存されない設計（安全側）。
   1. ユーザーのスマホで公式LINEを友だち追加→何かメッセージを1つ送ってもらう→「🔧セットアップ中です」と返ってくる
   2. あなたが実行：`curl -s -H "Authorization: Bearer $(cat .list_key)" "https://（URL）/enroll"` → userIdが返る
   3. そのuserIdを `wrangler secret put ALLOWED_USER` に設定（echoでパイプ）
   4. もう一度写真を送ってもらう→今度は「📦受け取りました」が返り、`/list` に載ること
   これで**他人が友だち追加して写真を送っても保存されない**
8. **秘密の後始末**：`.env`（作成場所は `worker/.env.tmp`・作成時に `chmod 600`）はsecret登録が全部終わったら**ファイルごと削除**。`chmod 600 .list_key` も実行
9. **なりすまし防止の確認**：`curl -s -o /dev/null -w "%{http_code}" -X POST https://（URL）/webhook -d '{}'` が **403** を返すこと
10. **完了通知の確認**：`curl -s -X POST -H "Authorization: Bearer $(cat .list_key)" -H 'content-type: application/json' -d '{"text":"セットアップ完了テスト🎉"}' https://（URL）/notify` でユーザーのLINEに届くこと（以降、完了報告はこの経路＝LINEトークンをPCに残さない）
